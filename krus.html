    Kruskal's Algorithm



    Kruskal's Algorithm Minimum Spanning Tree (MST):


    Kruskal's algorithm is a greedy algorithm used to find the Minimum Spanning Tree (MST) of a connected, weighted graph. The MST is a subset of the graph's edges that connects all vertices together without cycles and with the minimum possible total edge weight.



    HOW IT WORKS?

    Sort Edges by Weight: List all the edges in the graph and sort them in non-decreasing order based on their weights.

    Initialize MST: Create an empty data structure (e.g., a list or set) to store the edges of the MST.

    Union-Find Initialization: Use the Union-Find (Disjoint-Set) data structure to track which vertices are in the same connected component.

    1. Add Edges to MST:
      • Iterate through the sorted edges.
      • For each edge, check if its two vertices belong to different connected components.
      • If they do, add the edge to the MST and merge the two components using the union operation.
      • If they are already connected, skip the edge to avoid forming a cycle.
    2. Stop Condition: The algorithm terminates when the MST contains \( V - 1 \) edges, where \( V \) is the number of vertices.

    Advantages




      • Simplicity:Easy to understand and implement.Particularly effective for sparse graphs (graphs with fewer edges).


      • Optimal:Guarantees a minimum spanning tree if the graph is connected.


      • Independent of Starting Vertex:Unlike Prim's algorithm, it does not require starting from a specific vertex.


      • Parallelizable: Sorting edges can be parallelized, making it more suitable for distributed systems.


    Disadvantages




      • Sorting Overhead:The sorting step has a time complexity of 𝑂 ( 𝐸 log ⁡ 𝐸 ) O(ElogE), which can dominate the algorithm for dense graphs.


      • Union-Find Complexity:The efficiency depends on the implementation of the union-find data structure, requiring optimizations like path compression and union by rank.


      • Not Suitable for Dynamic Graphs: Changes in the graph (e.g., adding/removing edges) require recomputation from scratch.


      • Disconnected Graphs: The algorithm cannot handle disconnected graphs unless modifications are made.


    Applications



      • Network Design: Building network connections such as computer networks, electrical grids, and road systems while minimizing cost.


      • Clustering in Machine Learning: Used in hierarchical clustering to group data points.


      • Urban Planning: Laying pipelines or cables with minimum total cost.


      • Telecommunication:Optimizing the layout of communication lines between towers or stations.


      • Circuit Design: Used to connect components of an electronic circuit with minimal wiring.




    Pseudocode




    KRUSKAL_MST(graph):

    1.Initialize an empty list MST to store the edges of the Minimum Spanning Tree.

    2.Create a Union-Find data structure (Disjoint-Set) to track connected components.

    3. SORT all edges in graph by weight in non-decreasing order.

    4.FOR each edge (u, v) in the sorted edge list:



       IF UNION-FIND indicates that u and v belong to different sets:

            - Add edge (u, v) to MST.

       - UNION the sets containing u and v.

            (Else, skip the edge to avoid cycles).

       ELSE:

    5. RETURN MST.



    Explanation


    1.Sort all edges of the graph in non-decreasing order by weight.


    2.Start with an empty Minimum Spanning Tree (MST).


    3.Use a Union-Find data structure to check if adding an edge creates a cycle.


    4.Add the smallest edge that doesn’t form a cycle to the MST and merge the connected components.


    5.Repeat until the MST has 𝑉 − 1 V−1 edges, where 𝑉 V is the number of vertices.

    Time complexity




    Definition:Time complexity measures the amount of time Kruskal's algorithm takes to compute the Minimum Spanning Tree (MST) as a function of the size of the input graph, characterized by the number of vertices 𝑉 V and edges 𝐸 E. It reflects the cost of sorting the edges and performing Union-Find operations for cycle detection.



    Best-Case Time Complexity:𝑂 ( 𝐸 log ⁡ 𝐸 ) O(ElogE)


    Sorting the edges dominates, and the graph is sparse with minimal Union-Find operations.



    Average-Case Time Complexity:O(ElogE)


    Sorting and Union-Find operations occur as usual, with 𝛼 ( 𝑉 ) α(V) being nearly constant.



    Worst-Case Time Complexity:𝑂 ( 𝐸 log ⁡ 𝐸 ) O(ElogE)


    Sorting all edges and performing 𝐸 E Union-Find operations, even for dense graphs.


    Space complexity



    Definition:Space complexity measures the amount of memory Kruskal's algorithm requires to compute the Minimum Spanning Tree (MST) as a function of the size of the input graph. It accounts for storing the list of edges, the Union-Find data structure (parent and rank arrays), and the MST edges. The overall space depends on the number of vertices 𝑉 V and edges 𝐸 E.



    Space Complexity: 𝑂 ( 𝐸 + 𝑉 ) O(E+V)



    kruskal's algorithm requires space for storing all 𝐸 E edges for sorting, and additional 𝑂 ( 𝑉 ) O(V) space for the Union-Find data structure (parent and rank arrays), resulting in a total space complexity of 𝑂 ( 𝐸 + 𝑉 ) O(E+V).

    Summary:

    Kruskal's algorithm efficiently finds the Minimum Spanning Tree (MST) by sorting edges and using a Union-Find structure to avoid cycles. Its time complexity is dominated by edge sorting, and space complexity depends on storing edges and Union-Find data.
